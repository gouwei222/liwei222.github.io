一、资源位推荐

基础 ESMM 为什么不能直接训一个 CVR 模型？
两个致命问题： ① 选择偏差（Selection Bias）：CVR 只能用点击样本训练，但 inference 在曝光全集——训练空间 ≠ 推理空间。 ② 样本稀疏：曝光 >> 点击 >> 转化，CVR 可用样本极少，极易过拟合。 ESMM 解法：pCTR 和 pCTCVR 都可全量训练，pCVR = pCTCVR / pCTR 间接学习。CVR tower 共享 CTR 的 embedding。
基础 IPW 中 min(log(1/p), 20) 截断为什么？
逐层拆解： • 1/p：p=0.0001 → 权重=10000，一个样本主导 loss。 • log(1/p)：把 [1,∞) 压缩到 [0,∞)，10000→9.2。 • min(...,20)：硬截断到 20。 本质是 bias-variance tradeoff：截断引入少量 bias，但大幅降 variance。
基础 纠偏模型 AUC=0.7，为什么「不是越高越好」？
这个模型预测的是业务方的「分配规则」（propensity），不是用户真实偏好。AUC=0.95 说明完美复现了业务规则——但业务规则的产物（有偏曝光）正是要纠偏的对象。正确评估维度是 SMD 是否下降。
进阶 K-means 分簇 IPW vs 全量 IPW？聚类解决什么根因？
全量 IPW 掩盖组间异质性。A 类用户天然倾向活动 X，B 类倾向 Y——全量 IPW 会把 B 类中 Y 的低转化错误归因于「曝光不足」，赋予不合理高权重。分簇后每簇内用户行为同质，曝光差异更可能反映真实 bias，各簇互不污染。
进阶 X→T→Y 的 unconfoundedness 假设不成立怎么办？
①敏感性分析——量化 confounder 需要多强才能推翻结论。②工具变量 IV（活动排期自然轮换）。③DID + 自然实验。④双稳健估计 DR（propensity+outcome，一个正确就一致）。⑤在线 A/B 最终裁判。
高阶 离线只取第一条曝光，线上多次请求怎么办？
①实时特征更新（Kafka/Flink）——session 级特征（今日曝光/点击次数）。②频控+多样性。③Thompson Sampling 探索。④Platt scaling 分布校准（训练分布 ≠ 线上分布）。
高阶 冷启动活动零曝光怎么分配资源位？
方案一（属性相似度）：新活动属性 embedding → 最相似历史活动 CTR/CVR 作为先验。方案二（Thompson Sampling）：Beta(1,1) 宽先验，每次采样选最高值曝光，随时间收敛。方案三（UCB）：score + c×sqrt(log(t)/n)，不确定性项自动给探索机会。实际系统通常方案一+方案二混合。

二、电销坐席匹配

基础 XGB 建模 user×坐席 为什么不行？
三层根因：①XGB 是加性模型——user+坐席 效应被近似为两相加，组合效应丢失。②样本稀疏——每个 user×seat 只有几十条样本，树不会为此分裂。③叶子区域均值——同一 user 不同 seat 大概率落同一叶子→打分接近→差异度小。XGB 学到的本质是「用户动支意愿」，坐席差异被平均掉了。
基础 为什么不建金额回归而建二分类？
金额长尾分布——5% 用户贡献 80% 金额。MSE 对称惩罚：预测 100 实际 10000 的 error² 主导梯度。学好长尾→中间不准。业界通常建多个二分类，在分类序上看金额排序性。
进阶 Wide&Deep 为什么比 XGB 效果好？
①Wide 侧——手动交叉特征直接喂入，离散跳变，差异化天然大。②Deep 侧——embedding+MLP 学隐式交叉。③XGB 对高基数类别交叉效果差（笛卡尔积基数极高，分裂收益不够）。DeepFM 不用是因为 FM 侧学不了精细的业务逻辑交叉。
高阶 拉格朗日松弛在实时分配解决什么？
核心难题：序贯决策 vs 全局约束的耦合。分配 user_1 时不知道后面还有多少用户、什么质量。拉格朗日松弛把「坐席名单量均衡」从必须满足变成违反就罚款——每个坐席维护一个"价格" λ，拿超了价格涨，后面用户自然不选。类比高速收费站——每个司机看哪个口排队短往哪走，无需中央调度。

三、用户运营 Agent

基础 为什么需要「对话路由」？
三个失败 case：①「帮我写邮件」→槽位全空→LLM 硬凑维度→跑偏。②「加个渠道过滤」→不认为是补充→丢失上一轮条件。③两个 Skill 互不知道上下文→切换时重新描述。对话路由做三件事：意图分类、增量补充合并、跨槽位携带。
基础 判断新话题 vs 补充为什么用硬规则不用 LLM？
①规律足够固定（帮我/我想→新话题，加个/换成→补充），覆盖 90%+。②确定性 100% vs LLM 可能误判。③延迟微秒级 vs 秒级。但正则未命中时走 LLM 兜底——规则覆盖确定性场景，LLM 覆盖边缘场景。
进阶 五道 SQL 校验防线，只留三道你留哪三道？
①安全（底线，DROP/INSERT 绝不执行）；②白名单（幻觉是 Text-to-SQL 最大问题——语法正确但字段不存在=业务错误）；③逻辑（GROUP BY 完整性+分区过滤——最常见业务错误）。语法修起来成本低，EXPLAIN 可通过缩小日期绕过。
进阶 修正节点为什么强调「只修改有问题的部分」？
文档遇到过：修正节点被告知字段名不对→替换时误删 GROUP BY→引入新错误→再次修正→死循环。限定只改错误点缩小修正空间→稳定性提升。ReAct 的 diff 思路强化这点——先看清楚改了什么，再判断新问题是不是自己引入的。
进阶 为什么用槽位重叠度而不是语义检索匹配模板？
数据是结构化的（槽位=key-value），Python 算重叠度确定、毫秒、零成本。语义检索的坑：「老客动支金额按意愿分切效果好」vs「新客动支金额按意愿分切效果差」→embedding 极近但结论相反——一个该优先试一个该跳过。语义检索分不出好坏。
进阶 切分点评分：Lift × log(人群) × 覆盖度惩罚——为什么不能只看 Lift？
只看 Lift 会选「极高但极窄」的切分点——圈 50 个人 Lift=10，对运营没意义。三个因子：Lift 保证效果；log(人数)保证规模；覆盖度惩罚让占比倾向 30%。p<0.05 硬门槛。
高阶 RAG「经验只定顺序不替代验证」如何强制执行？
三层设计：①RAG 输出是 priority_dims+skip_dims+examples（表格，标注仅供参考）。②Python 引擎强制调 test_dimension 拉真实数据，不经过 LLM。③ReAct 拦截跨人群经验迁移——LLM 联想时先验证再输出。绕过→新客经验推老客→AB 无效→浪费预算。
高阶 Offer 用规则引擎、数字用 Python 模板——背后的架构原则？
同一个原则：能做硬约束的地方，绝对不让 LLM 自由发挥。Offer：LLM「额度高」可能忽略「尚未授信」→Python 硬编码映射。数字：LLM「凭记忆」写数字→记错就编→Python str.replace 填模板。LLM 只负责 qualitative 文案。边界：规则维护成本 > LLM成本+幻觉风险时，才放权。

四、超值出租车 & 预估价

基础 为什么拆 Base_ECR + Delta_ECR 两个模型？
决策公式需要独立判断：Base 低 Delta 高→最终还是低；Base 高 Delta 低→发券没意义。分开建模：Base 用全量数据更准；Delta 用 RCT 数据做无偏弹性估计。ROI = uplift × value - cost。
进阶 因果森林 vs 随机森林本质区别？诚实分割是什么？
随机森林预测 Y（发单概率），因果森林预测 τ = Y_T - Y_C（增量）。分裂标准不同——随机森林最小化 Y 方差，因果森林最大化 τ 在叶子间差异。诚实分割：数据分两半，一半决定分裂，一半估计叶子 effect——防止同一批数据既分裂又估计的 overfitting bias。输出 CATE。
基础 为什么选 MLP 不是 XGBoost 做预估价回归？
①预估价平滑连续——MLP 平滑，XGBoost 阶梯状。②120+ 特征交互复杂——MLP 隐层自动学，XGB 只能学低阶。③ReLU 不怕大误差——正半区梯度恒为 1，tanh/sigmoid 大误差时梯度消失。
基础 长单短单为什么分开建模？
MSE 对误差绝对值天然敏感。长单(100-300元)误差 20→MSE=400；短单(10-20元)误差 5→MSE=25。统一建模时模型更关注长单→短单被高估、长单被低估。分开后各自价格区间内公平竞争。

综合交叉

进阶 Agent vs 传统 ML 项目面试评估侧重点？
传统 ML 问「为什么选这个模型」，Agent 问「为什么不在这用 LLM」。Agent 工程师需要知道 LLM 的边界——能写规则不调 LLM，能校验不留隐患，能把失败转化为系统成长。
高阶 从五个项目中选最优区分度的深挖，追问链怎么设计？
选用户运营 Agent——展示最完整的问题解决框架。追问链：①「错了转化口径哪道防线拦住？」→考防线边界。②「100 人同时用最大瓶颈？」→考系统思维（从库并发）。③「什么时候该反过来用 LLM 优先？」→考辩证思维。④「运行一年后最大技术债务？」→考演进思维（few-shot 模板劣化）。
